## 🧠 Why This Repo

LeetCode is pattern recognition, not memorization. Every problem in this repo is solved with one goal:

**Understand the pattern deeply enough to apply it to a problem you've never seen.**

Patterns I focus on:

| Pattern | Examples |
|---------|----------|
| Sliding Window | Longest Substring, Max Sum Subarray |
| Two Pointers | Trapping Rain Water, Container With Most Water |
| Backtracking | Combination Sum, Permutations, Sudoku Solver |
| Binary Search | Search in Rotated Array, Find Minimum |
| Dynamic Programming | Climbing Stairs, Coin Change, LCS |
| Greedy | Jump Game, Gas Station |
| Graph DFS/BFS | Number of Islands, Clone Graph |
| Stack / Monotonic Stack | Valid Parentheses, Next Greater Element |
| Heap / Priority Queue | Top K Elements, Merge K Lists |
| Prefix Sums | Subarray Sum, Range Queries |

## 🧩 Problem-Solving Framework

Every problem follows the same mental model:

```
1. UNDERSTAND   → Read twice. Identify inputs, outputs, constraints.
2. BRUTE FORCE  → What's the naive O(n²) / O(n³) approach?
3. OPTIMIZE     → Can I use hashing? Sorting? Two pointers? DP?
4. PATTERN MAP  → Which known pattern does this resemble?
5. CODE CLEAN   → Readable > clever. Comments explain WHY, not WHAT.
6. ANALYZE      → State Time & Space complexity explicitly.
```
